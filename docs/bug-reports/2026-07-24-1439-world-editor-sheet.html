제외한 관찰

/favicon.ico 404는 WorldEditorSheet 흐름과 무관한 application shell 정적 자원으로 제외했다. conflict 검증 중의 world-entries PUT 409 console 항목은 의도적으로 만든 STORY_BIBLE_REVISION_CONFLICT 응답으로 결함에서 제외했다.

수행한 검사

  • git status --short --branch: 시작 시 기존 미추적 보고서·증거·문서를 기록하고 보존했다.
  • zellij-agent ticket-worker list --json: 등록 전·후 중복 확인, #15와 #16의 모든 값 및 ready 상태를 fresh JSON으로 확인했다.
  • 정상 흐름: 기존 수정, 새 항목 추가, blank title/description validation과 첫 invalid focus, authoritative save response·success announcement·Sheet close를 확인했다.
  • failure/recovery: world-entries PUT 일회성 network failure에서 draft 보존·alert·재시도 성공을 확인했다.
  • conflict/reload: MSW 선행 저장으로 revision 2를 만든 뒤 UI PUT 409, local draft 보존, latest reload 확인 및 server 값 채택을 확인했다.
  • unavailable: 계약상 STORY_BIBLE_NOT_FOUND 404에서 draft 보존, field/add/save disabled, read-mode recovery action을 확인했다.
  • close/focus/scroll: clean Escape close와 launch focus, dirty discard dialog, validation scroll와 fixed footer를 확인했다.
  • console/network: favicon 404 외 예상되지 않은 console error가 없었고, Story Bible GET/PUT 200·409·GET 200 순서를 확인했다.

제약 및 미확인 항목

Playwright-test MCP의 활성 도구에는 viewport resize가 없어 실제 page는 1280×720으로 고정됐다. 따라서 지정된 375×800 mobile layout, 정확한 1280×800 높이, mobile overlay/scroll/focus는 미확인이다. loading state는 요청 중 control freeze를 코드와 기존 응답 시간 사이에서 완전히 포착하지 못했으며, 실제 backend persistence는 제외된 MSW-only 범위라 확인하지 않았다. 제품 코드는 수정하지 않았다.
