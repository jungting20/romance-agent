티켓

최소 재현 절차
  1. MSW seed 상태에서 원고 본문 끝에 고유 표식을 키보드로 입력한다.
  2. 다음 원고 PUT 한 번을 실패시켜 저장 실패 alert와 원고 저장 다시 시도 버튼을 표시한다.
  3. 재시도 버튼을 클릭하거나 Tab으로 이동해 Enter로 실행한다.
  4. PUT 성공과 자동 저장됨을 확인한 뒤 활성 요소, 본문 값과 선택 인덱스를 확인한다.
기대 결과
재시도 버튼이 사라진 뒤 현재 원고 본문 textarea가 포커스를 받고, 기존 본문과 캐럿 또는 선택 범위를 유지해 즉시 집필을 이어 갈 수 있어야 한다.
실제 결과
본문과 캐럿 인덱스 및 PUT 성공은 보존되지만 document.activeElement === document.body가 되어 보이는 키보드 포커스가 사라진다.
2회 clean-state 증거
1차는 MSW session 저장소를 seed로 초기화한 뒤 [FAIL-B]를 입력하고 마우스로 재시도했다. 2차는 다시 seed로 초기화한 뒤 [FAIL-C]를 입력하고 Tab/Enter로 재시도했다. 두 실행 모두 자동 저장됨, 선택 인덱스 182..182, 고유 표식 보존과 동시에 활성 요소 BODY를 기록했다.
사용자 영향
저장 실패에서 회복한 키보드 사용자가 집필 위치를 잃고 문서 처음부터 다시 탭 이동해야 한다. 데이터는 보존되지만 핵심 편집 복구 흐름의 키보드 연속성과 보이는 포커스가 끊긴다.
요구사항
frontend/docs/frontend-coding-rules.md의 보이는 포커스·키보드 호환 요구와 원고 저장 실패 재시도 및 초안 보존 계약을 충족하지 못한다.
확인된 위치와 가설
frontend/src/pages/writing-workspace/writing-workspace-page.tsx의 AutosaveIndicator는 오류 상태에서만 재시도 버튼을 렌더링하고 성공 시 제거한다. 호출부에는 sceneNavigation.editorRef 포커스 복원이 없다. 조건부 버튼 제거가 body 이탈을 만든다는 설명은 소스 기반 가설이며 구현 테스트로 확정해야 한다.

중복된 관찰

확정된 중복 관찰은 없다. fresh ticket-worker list --json의 #1–#11과 기존 버그 보고서·설계·계획에서 재시도 성공 후 편집기 포커스 이탈 항목을 찾지 못했다. #1 스크롤 경계와 #2 reload 복원은 이번 정상 회귀 검사에서 통과한 별도 결함이다.

제외한 관찰

/favicon.ico 404 1건은 ManuscriptEditor 편집 흐름과 무관한 정적 자산 요청이며 사용자 흐름 영향이 없어 티켓에서 제외했다. 지정된 세 뷰포트는 도구 제한으로 재현 자체를 수행하지 못했으므로 결함으로 분류하지 않았다.

수행한 검사

  • 정상 입력: 실제 키 입력 뒤 편집 중, 원고 PUT 200, 자동 저장됨을 확인했다.
  • 선택과 캐럿: 키보드로 15자를 선택해 [SEL-A]로 교체했고 기존 본문과 교체 뒤 캐럿을 확인했다.
  • 빈 값과 장면 전환: 새 빈 장면은 값 "", 선택 0..0, editor focus로 열렸고, 두 장면의 본문은 전환 뒤 서로 섞이지 않았다.
  • 긴 원고: 80개 문단, 3,749자에서 문서 높이 720, 중앙 main scrollHeight=5982/clientHeight=656, header top 0을 기록했다.
  • reload 복원: 긴 본문과 추가 장면을 자동 저장한 뒤 같은 탭 reload에서 복원했다.
  • 실패·회복: 일회성 PUT 실패에서 alert와 이름 있는 재시도 버튼, 최신 원고 보존, 재시도 PUT 성공을 확인했다.
  • 접근성: textbox의 접근 가능한 이름 원고 본문, 장면 전환 포커스, 실패 alert, 키보드 재시도와 성공 후 활성 요소를 확인했다.
  • 콘솔·네트워크: 제품 오류 없이 favicon 404만 기록했고 workspace GET 200과 원고 PUT 200을 확인했다.
  • zellij-agent ticket-worker list --json: 등록 전 중복 검사와 등록 후 #12의 ready 및 모든 필드 일치를 확인했다.
  • .codex/agents/tests/validate-bug-hunter.sh: 보고서와 구조 검증에 사용했다.

제약 및 미확인 항목

전용 브라우저 도구가 1280x720로 고정되어 요청된 375x500, 1024x600, 1440x800 반응형·모바일 소프트 키보드 레이아웃은 미확인이다. 새 browser context를 만들 수 없어 각 재현 전 romance-agent:msw:manuscripts:v1 session 저장소를 지우고 route를 다시 열어 동일 seed 시작 상태를 만들었다. 실제 OS 소프트 키보드와 보조기술 음성 출력, backend persistence, revision 충돌은 확인하지 않았다. 실패는 페이지의 다음 원고 PUT 한 번만 거부해 재현했고 외부 네트워크는 사용하지 않았다.
