탐색 범위

WorldEditorInitializingSheet가 직접 URL에서 세계관 데이터를 처음 불러오는 흐름만 점검했다. MSW로 지연·성공·500 INTERNAL_ERROR·수동 재시도 성공·404 STORY_BIBLE_NOT_FOUND·빈 목록을 구성하고, Skeleton/status, clean close, Escape, Tab/Enter, 포커스 복원, 콘솔과 네트워크를 확인했다.

제외 범위

세계관 편집·추가·저장·dirty discard, 다른 Sheet 및 작업공간 컴포넌트, 제품 코드·테스트·설정 수정, 새 서버, 외부 네트워크, 실제 데이터 변경은 제외했다.

등록한 버그

신규 등록 가능한 확정 버그가 없습니다. 확인된 폭 결함은 같은 화면을 병행 탐색한 보고서의 동일 수리 단위와 중복되어 티켓, 버그별 설계 문서, 구현 계획 및 증거 자산을 새로 만들지 않았습니다.

중복된 관찰

1280x720에서 초기화·retryable error·unavailable 상태의 오른쪽 Sheet가 모두 실제 폭 384px로 렌더링됐다. UI 계획은 desktop Sheet를 약 40rem으로 규정한다. loading 상태와 별도 error clean 상태에서 두 번 이상 같은 폭을 확인했지만, 병행 WorldEditorSheet 보고서의 #bug-002, world-editor-desktop-width 설계·계획, Medium ready 티켓 #16 세계관 편집기 데스크톱 폭이 384px로 축소됨과 동일한 공용 Sheet class 충돌·동일 수리 범위이므로 중복으로 분류했다. 탐색 전에는 기존 티켓 #1–#13, 종료 전에는 #16을 포함한 fresh JSON을 확인했다.

제외한 관찰

초기 페이지의 /favicon.ico 404 콘솔 오류 1건은 WorldEditorInitializingSheet 및 지정 사용자 흐름과 무관해 제외했습니다. 의도적으로 구성한 Story Bible 500 2회와 404 1회는 오류 UI 검증용 MSW 응답이므로 결함으로 세지 않았습니다.

수행한 검사

  • 정상 경로 1회차: 직접 URL 성공 후 dialog 제목·설명·기존 항목과 하단 액션을 확인했고 첫 분류 combobox에 포커스가 이동했다.
  • 초기화 지연: MSW delay('infinite')에서 polite status의 “세계관을 불러오는 중이에요.”와 두 Skeleton 영역, 편집 컨트롤 부재, close 버튼 포커스, Escape clean close 및 URL의 panel 제거를 확인했다.
  • 지연 회복: pending query를 취소하고 기본 MSW 성공 응답으로 복구한 뒤 첫 기존 항목 분류 포커스를 확인했다.
  • retryable 오류: MSW가 500 INTERNAL_ERROR를 두 번 반환한 뒤 alert와 “다시 불러오기”를 확인했고, 수동 retry의 세 번째 200에서 편집기와 첫 필드 포커스가 복구됐다.
  • unavailable 오류: MSW 404 STORY_BIBLE_NOT_FOUND에서 전용 alert와 “세계관 보기로 돌아가기”를 확인했다. close에서 Tab으로 액션에 이동하고 Enter로 실행했을 때 ?tab=world로 닫히며 “세계관 보기” tab에 포커스가 돌아갔다.
  • 빈 경계: MSW의 빈 worldEntries 성공 응답에서 빈 안내, 추가 액션, disabled 저장, Sheet 제목 초기 포커스를 확인했다.
  • 정상 경로 2회차: clean MSW 기본 응답과 query cache 재생성 후 같은 직접 URL을 다시 열어 dialog, 기존 항목, 첫 분류 포커스, footer 도달성을 재확인했다.
  • 네트워크: Story Bible GET의 200, 의도한 500 2회, retry 200, 의도한 404, 회복 200을 Playwright 네트워크 로그에서 확인했다.
  • 서버 확인: curl로 대상 route가 HTTP 200을 반환함을 확인했다.
  • 공식 문서 검증: .codex/agents/tests/validate-bug-hunter.sh를 실행했으나, 보고서 검사 이전에 현재 .codex/agents/bug-hunter.toml의 필수 문구 누락으로 실패했다. 타인 소유 변경을 보존하기 위해 해당 파일은 수정하지 않았다.
  • 보고서 한정 정적 검사: UTF-8, lang="ko", viewport metadata, 필수 h2, zero-result 문구, 미해결 token 부재, script 부재를 별도 read-only Python 검사로 확인했다.

제약 및 미확인 항목

Playwright Test MCP의 현재 allowlist에는 viewport resize 도구가 없어 요청된 375x800 모바일과 정확한 1280x800을 실행하지 못했다. 따라서 모바일 full-width Sheet, 800px 높이, breakpoint별 focus/keyboard 동등성은 미확인이다. 지연·오류 시나리오는 공유 서버의 브라우저 MSW 런타임에서만 구성했으며 외부 네트워크와 실제 backend는 확인하지 않았다. 공식 validator는 저장소 agent 설정의 선행 실패로 보고서까지 도달하지 못했다. 이 보고서는 지정된 초기화 Sheet 흐름 밖의 애플리케이션 품질을 주장하지 않는다.
