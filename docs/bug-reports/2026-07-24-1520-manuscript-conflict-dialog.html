중복된 관찰

최초 및 등록 직전 ticket-worker list --json를 비교했고 이 네 결함과 동일한 티켓은 없었다. 기존 #12는 일반 자동 저장 재시도 성공 후 textarea 복원, #16/#18은 다른 Sheet의 폭 문제여서 중복이 아니다.

제외한 관찰

Escape 후 상단의 사라지는 충돌 해결 열기 버튼으로 재진입해 해결할 때 focus가 BODY에 남는 관찰은 한 번만 확인되어 등록에서 제외했다.

수행한 검사

  • scene-content와 scene-structure 정상 경로, local/server 해결, Escape 초안 보존, Tab 순환과 direct close focus restore를 확인했다.
  • 실제 MSW 404 compare failure 뒤 장면 복구와 비교 재시도 성공을 확인했다.
  • 1회 해결 PUT failure 뒤 오류/재시도 성공과 revision 증가를 확인했다.
  • 네 결함은 각각 최신 MSW revision을 다시 읽는 깨끗한 시작 상태에서 두 번 재현했다.
  • 네트워크는 workspace 200, 외부 PUT 200, stale PUT 409, scene-diffs 200/의도한 404 순서였고 예상 밖 API 요청은 없었다.
  • 콘솔에는 의도한 409/404 resource 오류와 favicon 404 외 별도 page exception이 없었다.

제약 및 미확인 항목

playwright-test MCP allowlist에 resize 도구가 없어 실제 세션은 1280×720이었다. 따라서 요청된 1280×800 높이와 375×800 모바일 layout은 fresh browser evidence로 확인하지 못했다. loading 문구는 요청이 빠르게 완료되어 정지 화면으로 확보하지 못했으나 compare error/retry와 structure ready 상태는 확인했다. 제품 원인은 구현 전 가설이며 제품 코드는 수정하지 않았다.
