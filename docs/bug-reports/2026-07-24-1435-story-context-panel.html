이 범위와 실제 검증 환경에서 확정된 버그가 없습니다. 따라서 신규 티켓, 버그별 설계 문서, 구현 계획, 스크린샷 증거는 생성하지 않았습니다.

중복된 관찰

티켓 큐를 fresh JSON으로 확인했으며 이 범위에서 중복으로 분류할 확정 결함은 없었다. 기존 #1 집필 에디터 스크롤 경계 수정은 완료 상태이며, 이번 카드 경계 관찰은 실제 조상 ScrollArea가 스크롤을 제공해 결함이 아니므로 중복 티켓으로 세지 않았다.

제외한 관찰

첫째, 인물 5카드 경계에서 active tabpanel 자체는 높이가 늘어났지만 조상 ScrollArea가 overflow-y: scroll과 scrollTop을 소유하고 Tab 이동으로 마지막 수정 버튼까지 노출되어 결함에서 제외했다. 둘째, favicon.ico 404는 StoryContextPanel 사용자 흐름 밖의 정적 자원 문제라 제외했다.

수행한 검사

  • 정상 경로 2회: 인물 2카드와 역할 Badge, 숨은 감정, 대상명을 포함한 수정 버튼, 세계관 장소 Badge·제목·설명을 확인했다.
  • 편집 진입 2회: 신규 인물 이름 포커스, 기존 인물 이름·읽기 전용 ID, 세계관 첫 분류 포커스와 내부 스크롤을 확인했다.
  • 키보드·포커스: Enter로 신규 편집기 열기, Escape clean close, 실행 버튼으로 포커스 복귀, 경계 카드 Tab 순서와 조상 스크롤 회복을 확인했다.

  • 실패·회복: 존재하지 않는 characterId deep link에서 alert, disabled form, 목록으로 돌아가기와 URL 회복을 확인했다.
  • 네트워크·콘솔: Story Bible GET 200 OK, 인물 POST 201 Created 3회. 앱 오류는 없었고 범위 밖 favicon.ico 404만 관찰했다.

제약 및 미확인 항목

요청된 375×800 모바일과 정확한 1280×800 데스크톱은 viewport 전환 제한으로 확인하지 못했다. clean silver-garden fixture에는 인물 2명과 세계관 1개가 있어 두 목록의 진짜 빈 상태를 만들 수 없었고, 제공 도구에는 MSW handler override나 network failure 주입 기능이 없어 읽기 실패·저장 실패·재시도는 미확인이다. 경계 스크롤은 인물 카드에서만 확인했으며 세계관 다중 카드 목록은 초기 fixture와 데이터 변경 최소화 제한 때문에 미확인이다. 이 문서는 위 범위와 실제 1280×720 환경의 0건 결과만 주장한다.
