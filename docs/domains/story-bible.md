# Story Bible 도메인 계약

## 목적과 책임

Story Bible은 작품 안에서 사실로 취급되는 인물과 세계관의 기준 정보를 소유한다. 집필 장면에 필요한 정보만 선택해 문맥으로 제공한다.

## 보편 언어

- **스토리 바이블:** 프로젝트에 속한 인물과 세계관 기준 정보의 모음
- **인물 카드:** 불변 식별자와 이름, 성별, 나이, 역할, 성격, 문체, 대사 스타일,
  기준 욕망, 숨은 감정을 가진 인물 기준 정보
- **기준 욕망:** 인물이 작품 전반에서 겉으로 이루고 싶어 하는 목표. 장면 현재 욕망이
  아니다.
- **숨은 감정:** 인물이 직접 드러내지 않지만 행동과 대사에 영향을 주는 마음
- **세계관 항목:** 장소, 사물, 규칙 중 하나로 분류되는 작품 속 기준 정보
- **장면 문맥 참조:** 장면과 관련된 인물 및 세계관 항목 식별자 목록
- **장면 문맥:** 유효한 참조에 해당하는 인물과 세계관 정보의 부분집합
- **확인된 관계 사건:** 사용자가 Narrative Memory 후보의 근거를 검토해 사실로 확인한 시간적 인물 관계
- **확인된 장소 사건:** 사용자가 Narrative Memory 후보의 근거를 검토해 사실로 확인한 인물의 도착,
  현재, 출발 사건
- **확인 사실 식별자:** Writing Assistant가 일관성 진단의 근거로 인용할 수 있는 Story Bible 사실의 고유
  식별자

## 핵심 모델

스토리 바이블은 프로젝트 식별자, 인물 목록, 세계관 항목 목록, 확인된 관계 사건과 장소 사건
목록으로 구성된다. 인물은 프로젝트 안에서 고유한 서버 생성 불변 식별자와 이름, 성별, 나이,
역할, 성격, 문체, 대사 스타일, 기준 욕망, 숨은 감정을 가진다. 이름을 제외한 문자열 속성은
빈 문자열일 수 있으며, 나이는 숫자로 변환하지 않는다. 세계관 항목은 고유
식별자와 분류, 제목, 설명을 가진다. 확인된 사건은 고유 확인 사실 식별자, 장면 순서, Narrative Memory
근거 참조를 가진다.

## 불변 조건

- 스토리 바이블은 하나의 프로젝트에 속해야 한다.
- 초기 스토리 바이블은 Story Design에서 검증한 두 주인공을 인물로 만들어야 한다.
- 인물과 세계관 항목은 프로젝트 안에서 식별 가능한 값을 가져야 한다.
- 인물 식별자는 서버가 생성하며, 등록 후 수정하지 않는다.
- 인물 이름은 앞뒤 공백을 제거한 뒤 비어 있어서는 안 된다.
- 인물 등록은 기존 인물과 세계관 항목을 보존하고 인물을 정확히 하나 추가해야 한다.
- 인물 수정은 하나 이상의 변경 속성을 받고, 생략한 속성과 다른 인물·세계관 항목의 값과 순서를
  보존해야 한다.
- 인물 등록과 수정은 항상 최신 저장 snapshot에 적용하고 Story Bible 리비전을 정확히 1 증가시켜야
  한다. 기대 리비전이나 충돌 해결 흐름을 사용하지 않는다.
- 서로 다른 인물의 수정은 서로를 덮어쓰지 않아야 하며, 같은 인물을 연속 저장하면 마지막으로 성공한
  저장이 최신 기준 정보가 되어야 한다.
- 세계관 항목의 제목과 설명은 앞뒤 공백을 제거한 뒤 비어 있어서는 안 된다.
- 기존 세계관 항목을 수정해도 그 항목의 식별자는 바뀌어서는 안 된다.
- 세계관 수정 요청에서 언급하지 않은 항목을 삭제하거나 변경해서는 안 된다.
- 장면 문맥에는 참조 식별자와 일치하는 인물과 세계관 항목만 포함해야 한다.
- 존재하지 않는 참조 식별자는 결과에서 제외해야 하며 기준 정보를 새로 만들거나 변경해서는 안 된다.
- 관계 사건과 장소 사건은 사용자가 근거를 검토하고 확인한 경우에만 Story Bible 사실이 되어야 한다.
- 미해결 후보와 `needs_review` 후보는 Story Bible 사실이 아니며 확정 문맥으로 제공해서는 안 된다.
- 확인 사실의 Narrative Memory 근거가 사라지면 해당 사실을 `needs_review`로 전환하고 재확인 전까지 확정 사실에서
  제외해야 한다.

## 유스케이스

### 초기 스토리 바이블 생성

프로젝트 식별자와 두 주인공 이름으로 초기 인물 정보와 첫 세계관 항목을 만든다.

### 장면 문맥 선택

장면이 참조하는 인물과 세계관 식별자를 받아 스토리 바이블에 존재하는 정보만 반환한다.

### 인물 카드 등록

식별자를 제외한 인물의 전체 기준 정보를 받아 이름을 정규화·검증하고, 프로젝트 안에서 고유한
불변 식별자를 생성해 최신 Story Bible에 인물을 하나 추가한다.

### 인물 카드 수정

기존 인물 식별자와 하나 이상의 변경할 기준 정보를 받아, 최신 Story Bible에서 해당 인물만 수정한다.
식별자는 유지하고 생략한 속성은 기존 값을 유지한다.

### Narrative Memory 후보 확인

명시적 애플리케이션 유스케이스가 사용자에게 Narrative Memory의 근거와 관계 사건 또는 장소 사건 후보를
제시한다. 사용자가 확인한 후보만 고유 확인 사실 식별자를 가진 Story Bible 사실로 등록한다.

### 세계관 항목 수정

기존 세계관 항목 식별자와 분류, 제목, 설명을 받아 식별자는 유지하고 나머지 정보를 변경한다. 존재하지 않는 식별자나 유효하지 않은 분류, 공백 제목 또는 공백 설명이 있으면 전체 변경을 거부한다.

### 세계관 항목 추가

분류, 제목, 설명을 받아 프로젝트 안에서 식별 가능한 새 세계관 항목을 만든다. 여러 수정과 추가를 한 번에 요청하면 모두 유효할 때만 전체 결과를 반환한다.

## 입력과 출력

- Projects에서 생성된 프로젝트 식별자를 입력받는다.
- Story Design에서 검증된 두 주인공 이름을 입력받는다.
- Manuscript 장면이 보유한 관련 인물 및 세계관 식별자를 입력받는다.
- 명시적 확인 유스케이스를 통해 Narrative Memory의 근거가 포함된 관계 사건과 장소 사건 후보를 입력받는다.
- 애플리케이션 유스케이스에서 수정할 기존 세계관 항목과 추가할 새 세계관 항목을 입력받는다.
- 인물 등록에서 이름, 성별, 나이 문자열, 역할, 성격, 문체, 대사 스타일, 기준 욕망, 숨은 감정을 입력받는다.
- 인물 수정에서 불변 인물 식별자와 변경할 속성만 입력받는다.
- 선택한 장면 문맥을 이를 요청한 애플리케이션 유스케이스에 제공한다.
- 확인된 관계 사건과 장소 사건을 확인 사실 식별자와 함께 일관성 검사 유스케이스에 제공한다.
- 수정과 추가가 모두 반영된 스토리 바이블을 이를 요청한 애플리케이션 유스케이스에 제공한다.
- 인물 등록·수정 후 전체 인물과 세계관 항목을 포함한 권위 있는 Story Bible snapshot과 증가한
  리비전을 제공한다.

## 책임지지 않는 영역

- 프로젝트 제목과 최근 활동 시각
- 이야기 트로프와 초기 콘셉트 검증
- 장면 순서와 원고 텍스트
- Narrative Memory 요약, 근거, 추출 후보, 분석 리비전, JSON 스냅샷
- 장면 현재의 욕망·의도와 알고 있는·모르는·알면 안 되는 정보 경계
- 인물 카드 안의 이전 Narrative Memory 저장. Narrative Memory는 필요한 이전 메모리를 durable
  `character_id`로 연결해 소유·제공한다.
- 집필 제안 생성과 적용
