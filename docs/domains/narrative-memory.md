# Narrative Memory 도메인 계약

## 목적과 책임

Narrative Memory는 Manuscript 장면에서 다시 만들 수 있는 장면 요약, 근거, 추출 후보, 분석 리비전,
버전이 있는 JSON 스냅샷을 소유한다. 후보를 작품의 사실로 취급하지 않으며 Manuscript나
Story Bible을 직접 변경하지 않는다.

## 보편 언어

- **장면 분석 리비전:** 특정 장면 본문 버전에 대해 수행한 분석의 버전
- **청크 분석:** 장면 본문의 유한한 부분에서 추출한 요약과 후보
- **근거:** 후보를 뒷받침하는 청크 식별자, 문자 범위, 원문
- **추출 후보:** 확인 전의 인물, 장소, 관계 사건, 장소 사건 해석
- **후보 상태:** `pending`, `approved`, `rejected`, `needs_review` 중 하나인 검토 상태
- **관계 사건:** 장면 순서와 근거를 가진 두 인물 사이의 시간적 사건
- **장소 사건:** 장면 순서와 근거를 가진 인물의 도착, 현재, 출발 사건
- **장면 관계 스냅샷:** 한 장면 리비전의 요약과 추출 후보를 담은 불변 JSON 구조
- **프로젝트 관계 스냅샷:** 현재 활성 장면 리비전들을 집계한 버전 있는 불변 JSON 구조

## 핵심 모델

- `Evidence`는 청크 식별자, 시작과 끝 문자 위치, 근거 원문을 가진다.
- `EntityCandidate`와 `PlaceCandidate`는 정규화 이름, 표시 이름, 별칭, 상태, 장면 리비전, 근거를 가진다.
- `RelationshipEventCandidate`와 `LocationEventCandidate`는 시간적 사건의 대상, 설명, 장면 순서,
  신뢰도, 상태, 근거를 가진다.
- `ChunkAnalysis`는 하나의 청크 요약과 추출 후보를 묶는다.
- `SceneRelationshipSnapshot`은 하나의 장면 리비전을, `ProjectRelationshipSnapshot`은 프로젝트의
  현재 활성 장면 리비전과 추출 후보를 불변 튜플로 보관한다.

## 불변 조건

- 모든 추출 결과는 후보이며, 사용자 확인 없이 자동으로 사실이 되어서는 안 된다.
- 모든 모음 필드는 튜플이어야 하며 스냅샷은 생성 후 변경할 수 없어야 한다.
- 빈 프로젝트 관계 스냅샷은 스냅샷 버전 `0`과 빈 모음을 가져야 한다.
- 재분석은 변경된 장면의 `pending` 후보를 해당 장면의 새 리비전 후보로 대체해야 한다.
- 재분석 후 근거가 사라진 확인 사실은 `needs_review`로 전환해야 하며 재확인 전에는 확정 사실로
  소비되어서는 안 된다.
- 관계와 장소는 정적 속성이 아니라 장면 순서에 따른 시간적 관계 사건과 장소 사건으로 표현해야 한다.

## 유스케이스

### 장면 분석

Manuscript에서 전달된 특정 장면 리비전의 본문을 유한한 청크로 나누고 요약, 근거, 추출 후보를
생성해 장면 관계 스냅샷을 만든다.

### 장면 재분석

장면 리비전이 바뀌면 이전 `pending` 후보를 대체하고, 기존 확인 사실의 근거가 사라졌다면
`needs_review` 전환이 필요함을 명시적 애플리케이션 유스케이스에 알린다.

### 프로젝트 스냅샷 구성

장면 순서와 활성 리비전에 따라 장면 스냅샷을 집계해 새 버전의 프로젝트 관계 스냅샷을 만든다.

### 후보 검토와 사실 확인

후보와 근거를 검토 흐름에 제공한다. 사용자가 확인하면 명시적 애플리케이션 유스케이스가 Story Bible에
확인된 관계 사건 또는 장소 사건 사실을 등록하도록 조정한다. Narrative Memory는 Story Bible을 직접
변경하지 않는다.

## 입력과 출력

- Manuscript의 프로젝트 식별자, 장면 식별자, 장면 순서, 장면 리비전, 장면 본문을 명시적
  애플리케이션 유스케이스를 통해 입력받는다.
- 근거가 포함된 인물, 장소, 관계 사건, 장소 사건 후보와 장면 요약을 반환한다.
- 불변 장면 관계 스냅샷과 버전이 있는 프로젝트 관계 JSON 스냅샷을 반환한다.
- Writing Assistant의 명시적 일관성 검사 유스케이스에 필요한 범위로 제한한 장면 요약을 제공한다.

## 책임지지 않는 영역

- Manuscript 장면, 장면 순서, 장면 리비전, 원고 본문의 소유와 변경
- Story Bible 사실의 직접 생성, 변경, 삭제
- 후보의 자동 승인 또는 자동 사실 전환
- Writing Assistant 제안 또는 일관성 진단 생성
- HTTP API, 백그라운드 큐, 저장소, 데이터베이스, 외부 모델 및 제공자의 선택
