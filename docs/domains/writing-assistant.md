# Writing Assistant 도메인 계약

## 목적과 책임

Writing Assistant는 사용자가 명시적으로 요청한 집필 지원 작업과 그 결과인 제안 또는 구조화된 대화 생성
결과를 소유한다. 원고를 자동으로 읽거나 변경하지 않으며, 제안 또는 생성 대화의 적용 여부는 이 도메인
밖에서 결정한다.

## 보편 언어

- **집필 지원 요청:** 작업 종류와 현재 장면 문맥을 결합한 명시적 요청
- **이어 쓰기:** 현재 감정선을 유지하는 다음 문단 삽입 제안
- **문장 다듬기:** 사용자가 선택한 문장을 다른 표현으로 교체하는 제안
- **대사 제안:** 인물 이름과 현재 문맥을 활용한 대화 삽입 제안
- **대화 장면 생성:** 세계관, 인물별 설정과 정보 경계, 현재 장면 문맥과 목적을 사용해 여러 발화와
  장면 결과 metadata를 만드는 명시적 작업
- **관련 이전 기억:** 호출자가 현재 대화 생성에 필요하다고 선택해 전달한 비권위 문맥. Story Bible의
  확인 사실을 대체하지 않는다.
- **정보 경계:** 인물별 현재 욕망, 숨긴 의도, 알고 있는 정보, 모르는 정보, 현재 공개 금지 정보와
  미래 공개 가능 시점의 모음
- **공개 금지 정보:** 현재 생성 시도에서는 실제 대사에 노출할 수 없고 미래 공개 조건만 문맥으로
  제공되는 안정적인 `information_id`의 정보
- **대화 생성 시도:** 하나의 원본 `scene_id`에 대해 고유 `generation_id`로 식별되는 모델 호출
- **대화 발화:** 고유 `turn_id`, 화자와 대상의 `character_id`·이름, 대사, 전달 방식, 행동과 subtext를
  가진 구조화 항목
- **일관성 검사:** 원고를 변경하지 않고 인물, 장소, 시간, 감정선의 문제를 알리는 진단
- **확인 사실 인용:** 일관성 진단이 근거로 사용한 Story Bible 관계 사건 또는 장소 사건의 고유
  확인 사실 식별자
- **제한된 Narrative Memory 요약:** 사용자가 명시적으로 요청한 일관성 검사에 필요한 장면 범위로 제한한
  재생성 가능한 요약
- **제안:** 삽입, 교체, 진단 중 하나의 적용 방식을 가진 결과

## 핵심 모델

집필 지원 요청은 작업 종류, 장면 본문, 선택한 텍스트, 인물 이름 목록으로 구성된다. 제안은 제안 식별자,
요청 작업, 적용 방식, 제목, 내용으로 구성된다.

대화 장면 생성 요청은 세계관, 인물별 `character_id`·이름·성별·나이·문체·대사 스타일·관련 이전 기억,
현재 `scene_id`와 문맥, 장면 목적, 인물별 정보 경계로 구성된다. 결과는 원본 장면과 생성 시도 상태,
순서가 보존된 대화 발화, 장면 요약·목적 결과·인물 상태·정보 흐름·미해결 긴장·다음 장면 훅 metadata로
구성되는 strict JSON 데이터다. `information_id`는 생성 시도 사이에서도 같은 정보를 안정적으로 가리킨다.

지원 작업은 이어 쓰기, 문장 다듬기, 대사 제안, 일관성 검사, 대화 장면 생성이다. 적용 방식은 삽입,
교체, 진단이며 대화 장면 생성은 별도의 구조화 JSON 결과를 반환한다.

## 불변 조건

- 사용자의 명시적 요청이 있어야 제안을 생성해야 한다.
- 문장 다듬기는 앞뒤 공백을 제거한 선택 텍스트가 있을 때만 요청할 수 있다.
- 이어 쓰기와 대사 제안은 삽입 제안을 반환해야 한다.
- 문장 다듬기는 교체 제안을 반환해야 한다.
- 일관성 검사는 진단 제안을 반환해야 하며 원고를 변경해서는 안 된다.
- 일관성 진단은 Story Bible의 확인된 관계 사건과 장소 사건만 사실 근거로 사용해야 한다.
- 일관성 진단이 Story Bible 사실을 근거로 삼으면 해당 확인 사실 식별자를 반드시 인용해야 한다.
- Narrative Memory의 미해결 또는 `needs_review` 후보를 확정 사실로 취급해서는 안 된다.
- Narrative Memory 요약은 명시적 일관성 검사에 요청된 장면 범위로만 제한해 소비해야 한다.
- 제안 생성은 전달받은 장면 본문을 직접 변경해서는 안 된다.
- 인물 이름이 부족하면 일반 역할명으로 대체할 수 있어야 한다.
- 대화 장면 생성은 전달받은 현재 장면 문맥과 원고를 직접 변경해서는 안 된다.
- 대화 장면 생성 결과의 `scene_id`와 목적은 요청과 같아야 하며 `generation_id`는 이번 생성 시도를,
  `character_id`는 인물을, `turn_id`는 결과 안의 발화를 식별해야 한다.
- 대화에 등장하는 화자와 대상은 요청에 포함된 인물 식별자와 이름을 정확히 참조해야 한다.
- 인물은 자신이 모르는 정보를 아는 것처럼 대사하거나 행동해서는 안 된다.
- 현재 공개 금지 정보는 미래 공개 가능 시점과 관계없이 이번 생성의 실제 대사에 직접 또는 쉽게
  추론 가능한 형태로 노출되어서는 안 된다. 공개 시점 충족 여부는 현재 생성 에이전트가 판단하지 않는다.
- 모든 공개 금지 `information_id`는 정보 흐름의 withheld 항목에 포함되어야 하고, 성공한 결과의
  `forbidden_information_violations`는 비어 있어야 한다. 누출이 탐지되거나 보고된 결과는 전체를 거부한다.
- 관련 이전 기억은 대화 연속성을 위한 비권위 문맥이며 Story Bible의 확인 사실이나 인물의 현재
  `known_information`을 덮어쓸 수 없다.

## 유스케이스

### 집필 제안 생성

요청 작업과 전달받은 문맥을 사용해 삽입, 교체, 진단 중 하나의 제안을 반환한다. 필요한 선택 텍스트가 없으면 문장 다듬기 요청을 거부한다.

### 명시적 일관성 검사

사용자가 요청한 장면 범위의 원고, Story Bible의 확인된 관계 사건과 장소 사건, Narrative Memory의
제한된 장면 요약을 사용해 진단 제안을 반환한다. 확인된 사실을 근거로 제시하는 각 진단은 해당 확인 사실
식별자를 인용한다.

### 대화 장면 생성

사용자가 명시적으로 요청하면 애플리케이션 유스케이스가 선택한 세계관·인물·관련 이전 기억과 현재 장면
문맥, 목적, 인물별 정보 경계를 전달한다. 에이전트는 한 번의 생성 시도에서 공개 금지 정보를 숨긴
구조화 대화와 검증 가능한 정보 흐름 metadata를 반환한다. 실패하거나 정보 경계를 위반한 결과는 부분
대화 없이 전체를 거부한다.

## 입력과 출력

- Manuscript의 현재 장면 본문을 애플리케이션 유스케이스를 통해 입력받는다.
- 선택된 원고 텍스트와 Story Bible에서 선택한 인물 이름을 입력받는다.
- 명시적 일관성 검사에 Story Bible의 확인된 관계 사건과 장소 사건, Narrative Memory의 제한된 장면
  요약을 입력받는다.
- 삽입, 교체, 진단 제안을 애플리케이션 유스케이스에 반환한다.
- 확인 사실을 근거로 삼은 진단에 Story Bible 확인 사실 식별자를 함께 반환한다.
- 삽입과 교체 제안은 사용자가 적용을 요청한 경우에만 Manuscript로 전달된다.
- 명시적 대화 장면 생성에는 Story Bible에서 선택한 세계관과 인물 설정, Manuscript의 원본 scene ID와
  현재 장면 문맥, 호출자가 선택한 관련 이전 기억과 인물별 정보 경계를 입력받는다.
- 대화 생성 결과는 JSON으로 애플리케이션 유스케이스에 반환되며 원고에 자동 적용하지 않는다.

## 책임지지 않는 영역

- 원고 텍스트의 소유와 직접 변경
- 장면의 커서 및 선택 범위 유효성 검증
- 인물과 세계관 기준 정보 관리
- Story Bible 관계 사건과 장소 사건의 확인 및 `needs_review` 상태 관리
- Narrative Memory 요약, 근거, 추출 후보, 분석 리비전, JSON 스냅샷 관리
- 자동 실행, 백그라운드 원고 분석, 제안 자동 적용
- 관련 이전 기억의 생성·저장·권위 판정, 공개 가능 시점의 자동 판정
- 대화 생성 결과의 자동 원고 적용과 생성 이력 저장
