# 원고 자동 저장 재시도 포커스 복원 설계

> 버그 보고서: [ManuscriptEditor 브라우저 탐색 보고서](../../bug-reports/2026-07-24-1419-manuscript-editor.html#bug-001)

## 문제

집필 화면에서 원고 자동 저장이 실패하면 헤더에 `저장 실패` 경고와
`원고 저장 다시 시도` 버튼이 나타난다. 사용자가 이 버튼을 활성화해 저장이 성공하면
버튼이 `자동 저장됨` 상태로 교체되지만 키보드 포커스는 원고 편집기로 돌아가지 않고
`document.body`로 이탈한다.

MSW `silver-garden` seed를 초기화한 두 실행에서 같은 현상을 확인했다. 1차는 버튼을
클릭했고, 2차는 Tab으로 버튼에 이동한 뒤 Enter로 실행했다. 두 실행 모두 원고 값과
캐럿 인덱스는 보존되고 PUT 재시도도 성공했지만, 완료 뒤
`document.activeElement === document.body`였다.

## 영향과 심각도

심각도는 **Medium**이다. 저장 실패에서 회복한 키보드 사용자는 편집 위치가 데이터로
남아 있어도 포커스를 잃으므로 문서 처음부터 다시 탭 이동해야 한다. 저장 복구 직후
집필을 이어 가는 핵심 흐름의 키보드 연속성과 보이는 포커스가 끊긴다.

## 요구사항 근거

- `frontend/docs/frontend-coding-rules.md`는 의미 있는 HTML, 접근 가능한 이름,
  보이는 포커스와 키보드 호환 상호작용을 요구한다.
- `docs/superpowers/specs/2026-07-16-manuscript-scene-add-design.md`는 저장 실패에서
  명시적인 재시도 버튼을 제공하고 원고 초안을 보존하도록 정한다.
- `docs/domains/manuscript.md`의 원고 본문과 텍스트 범위 불변 조건은 재시도 포커스
  처리 중에도 본문과 선택 범위를 변경하지 않아야 함을 뜻한다.

## 목표

- `원고 저장 다시 시도`를 마우스 또는 키보드로 실행해 저장이 성공하면
  `원고 본문` textarea로 포커스를 복원한다.
- 실패 전에 보존된 캐럿 또는 선택 범위를 그대로 유지해 즉시 집필을 재개할 수 있게 한다.
- 기존 `편집 중` → `저장 중` → `자동 저장됨` 상태와 원고 PUT, 재시도 오류 처리,
  장면 전환 포커스 규칙을 변경하지 않는다.

## 범위

### 포함

- 집필 워크스페이스의 자동 저장 재시도 성공 포커스 복원
- 마우스와 키보드 활성화 회귀 테스트
- 원고 값, 캐럿/선택 범위, 접근 가능한 상태 이름과 재시도 요청의 회귀 검증

### 제외

- 자동 저장 debounce, revision, 충돌 해결 의미 변경
- 다른 경고·대화상자·세계관·인물 편집기의 포커스 정책
- Manuscript 도메인, API, MSW 저장 형식 변경
- 반응형 레이아웃이나 스타일 재설계

## 설계

`LoadedWritingWorkspace`가 이미 `ManuscriptEditor`에 전달하는
`sceneNavigation.editorRef`를 재사용한다. 재시도 요청 성공이 확인된 뒤 현재 활성 장면의
`원고 본문` textarea에 포커스를 이동한다. 새 포커스 가능한 래퍼나 전역 DOM 검색을
추가하지 않는다.

재시도 함수는 성공 여부를 호출자가 관찰할 수 있어야 한다. 기존 저장 직렬화와 상태 전이는
그대로 두고, 성공한 경우에만 편집기 포커스를 복원한다. 재시도가 다시 실패하거나 충돌로
전환되면 오류 또는 충돌 조작을 위한 현재 UI를 유지하고 편집기로 강제 이동하지 않는다.

React 렌더 사이클에서 `원고 저장 다시 시도` 버튼이 제거되기 전에만 `focus()`를 호출하면
다시 `body`로 이탈할 수 있으므로, 성공 상태 렌더 이후 살아 있는 editor ref를 대상으로
복원해야 한다. 정확한 구현 방식은 테스트로 관찰 가능한 결과를 보장하는 범위에서 선택한다.

## 수용 기준

1. 저장 실패 뒤 `원고 저장 다시 시도`를 클릭해 성공하면 `원고 본문`이 포커스를 가진다.
2. 동일 버튼에 Tab으로 이동해 Enter로 실행해도 성공 뒤 `원고 본문`이 포커스를 가진다.
3. 재시도 전 원고 값과 캐럿 또는 선택 범위는 성공 뒤 변경되지 않는다.
4. 재시도 PUT은 한 번 실행되고 성공 상태는 접근 가능한 `자동 저장됨`으로 노출된다.
5. 재시도가 다시 실패하거나 충돌하면 기존 alert와 복구 동작을 유지한다.
6. 장면 추가·전환과 일반 자동 저장의 기존 포커스 및 저장 동작에 회귀가 없다.

## 확인된 구현 위치와 가설

`frontend/src/pages/writing-workspace/writing-workspace-page.tsx`의
`AutosaveIndicator`는 `status === "error"`일 때만 재시도 버튼을 렌더링하며,
성공 상태에서는 버튼을 제거한다. 같은 파일의 호출부에는 재시도 완료 뒤
`sceneNavigation.editorRef`로 포커스를 복원하는 처리가 없다. 조건부 버튼 제거가 브라우저의
포커스를 `body`로 되돌리는 직접 원인으로 보이지만, 이는 구현 전 테스트로 확정할 가설이다.
