# 사용자 승인이 필요한 기능

## 목적

이 문서는 Muse에서 사용자의 명시적 승인 없이 실행해서는 안 되는 기능과 승인 방식을 정의한다.
승인의 목적은 확인창을 늘리는 것이 아니라, AI의 원고 접근, 작품의 기준 정보 변경, 되돌리기 어려운 변경,
외부 공유처럼 사용자의 통제권이 필요한 경계에서 의도를 확인하는 것이다.

승인이 필요하지 않은 기능은
[사용자 추가 승인이 필요하지 않은 기능](./user-approval-not-required-features.md)을 따른다.

## 승인 수준

### A1. 명시적 실행

사용자가 의미가 분명한 버튼이나 메뉴를 선택하면 그 동작 자체를 승인으로 본다. 별도의 확인창은 표시하지
않는다.

예: `이어 쓰기`, `이 장면 분석`, `원고에 적용`.

### A2. 영향 확인

삭제, 덮어쓰기, 대량 변경처럼 현재 상태를 잃을 가능성이 있는 작업은 대상과 영향을 보여준 뒤 확인을
받는다. 가능한 경우 실행 취소 또는 복구 경로도 함께 제공한다.

### A3. 데이터·권한 동의

원고나 개인 데이터를 외부 시스템에 전송하거나 다른 사람에게 공개하는 작업은 데이터 범위, 대상, 목적,
권한을 설명하고 명시적 동의를 받는다. 동의는 철회할 수 있어야 한다.

## 추가 확인창이 필요하지 않은 승인

다음 기능은 명확한 실행 버튼 자체가 승인이다. 버튼을 누른 뒤 다시 확인하는 모달을 표시하지 않는다.

- AI 제안 생성 및 재생성
- 개별 Narrative Memory 분석과 후보 검토
- AI 제안의 `원고에 적용`
- 사용자가 선택한 범위의 일관성 검사

이 기능들은 결과를 자동 확정하지 않고, 적용 후 실행 취소나 검토 경로를 제공하는 것으로 안전성을
확보한다.

## 승인 없이 금지되는 자동 동작

- AI가 사용자의 요청 없이 원고를 읽거나 제안을 생성하는 행위
- AI 제안 또는 일관성 진단이 원고를 자동 변경하는 행위
- Narrative Memory 후보를 자동으로 Story Bible 사실로 등록하는 행위
- 원고 변경을 이유로 기존 Story Bible 사실을 자동 삭제하는 행위
- 작품을 외부 서비스나 다른 사용자에게 자동 공개하는 행위
- 동의 범위를 넘어 원고 또는 개인 데이터를 저장하거나 제3자에게 전송하는 행위

## 공통 UX 요구사항

- 승인 문구는 대상과 결과를 구체적으로 설명해야 한다.
- 파괴적 작업의 기본 선택은 취소여야 한다.
- 승인 버튼은 `확인` 대신 `프로젝트 삭제`, `이 버전으로 복원`처럼 실제 결과를 표현해야 한다.
- 처리 중에는 중복 실행을 막고 완료·실패 결과를 알려야 한다.
- 실패한 작업을 성공한 것으로 보이게 하거나 일부만 적용한 상태를 숨겨서는 안 된다.
- 가능한 모든 변경에는 실행 취소, 복구함 또는 버전 복원 중 하나를 제공한다.
