# 사용자 추가 승인이 필요하지 않은 기능

## 목적

이 문서는 사용자가 일반적인 입력, 탐색 또는 설정을 통해 이미 의도를 표현했으므로 별도의 확인창이나
2차 승인을 요구하지 않는 기능을 정의한다. 목표는 사용자의 통제권을 유지하면서 반복적인 확인으로 집필
흐름을 방해하지 않는 것이다.

AI 접근, 기준 사실 확정, 파괴적 변경, 외부 공유 등 승인이 필요한 예외는
[사용자 승인이 필요한 기능](./user-approval-required-features.md)을 따른다.

## 기본 원칙

- 사용자가 직접 입력하거나 명확한 버튼을 누른 일반 작업은 그 상호작용으로 의도가 충분히 표현된다.
- 별도 확인은 생략하되 저장 상태, 결과, 실행 취소 또는 복구 경로를 제공한다.
- 현재 작품의 기준 정보나 외부 공개 상태를 바꾸지 않는 조회·탐색 기능은 즉시 실행한다.
- 자동 기능은 사용자가 이미 만든 변경을 안전하게 보존하거나 파생 정보를 표시하는 범위에 머문다.

## 기능 분류

### 작품과 원고 구성

| 기능                          | 실행 방식                                | 필요한 보호 장치                     |
| ----------------------------- | ---------------------------------------- | ------------------------------------ |
| 새 프로젝트 생성              | 설정 폼의 `작업 공간 열기` 선택으로 실행 | 중복 제출 방지와 실패 시 입력 보존   |
| 새 장·장면 추가               | 추가 버튼 선택 즉시 생성                 | 직후 이름 수정과 실행 취소 제공      |
| 장·장면 제목 수정             | 입력 후 저장 또는 자동 저장              | 저장 상태 표시                       |
| 장면 순서 변경                | 드래그 또는 이동 메뉴로 즉시 반영        | 실행 취소와 키보드 대체 조작 제공    |
| 장면을 다른 장으로 이동       | 목적지를 선택하면 실행                   | 이동 결과 표시와 실행 취소 제공      |
| 장면 복제                     | 복제 메뉴 선택으로 실행                  | 복제본임을 구분하고 새 식별자를 부여 |
| 장면 상태 변경                | 구상·초안·퇴고·완료 중 선택              | 필터와 표시 상태 즉시 갱신           |
| 인물 추가 및 일반 수정        | 편집 폼 저장으로 실행                    | 유효성 검사와 저장 실패 시 입력 보존 |
| 세계관 항목 추가 및 일반 수정 | 편집 폼 저장으로 실행                    | 유효성 검사와 동시 수정 충돌 처리    |
| 작품 제목·로그라인 수정       | 편집 내용 저장으로 실행                  | 서재와 작업공간에 동일하게 반영      |
| 표지·트로프·장르 태그 수정    | 설정 저장으로 실행                       | 변경 결과 미리보기 또는 즉시 반영    |

삭제나 대량 덮어쓰기는 이 문서의 범위가 아니며 별도 승인이 필요하다.

### 자동 저장과 복구 기반

| 기능                | 실행 방식                                 | 필요한 보호 장치                                |
| ------------------- | ----------------------------------------- | ----------------------------------------------- |
| 원고 자동 저장      | 사용자의 편집 이후 자동 실행              | 저장 중·완료·실패 상태 표시                     |
| 자동 버전 스냅샷    | 저장 정책에 따라 자동 생성                | 보존 정책과 복원 가능한 버전 표시               |
| 로컬 복구 초안 보존 | 저장 전후 자동 실행                       | 서버 저장과 로컬 보존 상태를 혼동하지 않게 표시 |
| 자동 백업           | 사용자가 선택한 저장소와 주기에 따라 실행 | 최초 외부 저장소 연결은 별도 동의 필요          |
| 저장 실패 재시도    | 자동 재시도 정책 또는 `다시 시도` 선택    | 중복 저장과 조용한 데이터 손실 방지             |
| 실행 취소·다시 실행 | 명령 선택 즉시 실행                       | 적용 범위와 되돌릴 수 없는 경계 표시            |
| 버전 비교           | 비교할 두 버전을 선택하면 실행            | 두 버전의 식별 정보와 차이를 명확히 표시        |

이 표의 자동 기능은 사용자가 직접 만든 변경을 보존하는 범위에 한정한다. AI 분석이나 외부 데이터 전송은
자동 저장과 같은 묵시적 승인으로 처리하지 않는다.

### 조회와 탐색

- 작품 서재와 최근 작품 조회
- 장·장면·인물·세계관 화면 전환
- 마지막 작업 장면으로 이동
- 원고 전체 검색
- 최근 장면과 즐겨찾기 조회
- 북마크 추가·제거
- 키보드 명령 팔레트 사용
- 글자 수와 집필 진행률 조회
- 집필 통계와 목표 달성 현황 조회
- 플롯, 감정선, 관계 변화의 시각화 조회
- 승인된 Story Bible 사실과 근거 조회
- AI 제안 및 진단 이력 조회

조회 결과가 잘못되었거나 오래된 경우 사용자가 출처와 갱신 시점을 확인할 수 있어야 한다.

### 개인용 내보내기

- DOCX, PDF, TXT, Markdown 파일 생성
- 전체 작품 또는 선택한 장·장면 다운로드
- 개인용 백업 파일 다운로드
- 인쇄용 미리보기

다운로드 버튼을 선택한 행위로 의도가 충분히 표현되므로 별도 확인창은 필요하지 않다. 외부 플랫폼에
게시하거나 다른 사람에게 파일을 전송하는 작업은 개인용 내보내기가 아니며 별도 승인이 필요하다.

### 온보딩과 개인 설정

- 첫 프로젝트 안내와 도움말 열람
- 샘플 프로젝트 열람
- 빈 화면의 다음 행동 안내
- 테마, 글꼴, 원고 폭, 줄 간격 변경
- 키보드 단축키 설정
- 집필 목표와 마감일 설정
- 알림 표시 방식 변경
- 기본 내보내기 형식 선택

개인 설정이 원고 데이터를 외부로 보내거나 공개 범위를 변경한다면 해당 부분만 별도 승인을 받아야 한다.

## 별도 확인 대신 제공할 피드백

- 자동 저장 상태와 마지막 성공 시각
- 추가·이동·복제 완료를 알리는 짧은 상태 메시지
- 실행 취소 동작
- 실패 원인과 다시 시도 방법
- 입력값이 유효하지 않을 때 필드 단위 오류
- 비어 있는 목록에서 다음으로 가능한 행동

## 승인이 필요한 기능으로 전환되는 조건

다음 조건 중 하나라도 발생하면 이 문서의 기능이라도
[사용자 승인이 필요한 기능](./user-approval-required-features.md)의 승인 방식을 적용한다.

- 기존 원고나 기준 정보를 삭제하거나 덮어쓴다.
- 여러 항목을 한 번에 되돌리기 어렵게 변경한다.
- AI 또는 외부 제공자에게 원고나 개인 데이터를 새로 전달한다.
- Narrative Memory 해석을 작품의 확정 사실로 등록한다.
- 다른 사용자 또는 외부 플랫폼에 작품을 공개하거나 전송한다.
- 사용자가 선택한 범위보다 넓은 데이터나 권한을 사용한다.
